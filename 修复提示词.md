【升级 1：Engine.js 支持 Beat 条件分支】

当前 beats 数组不支持 condition，导致后续剧情无法根据 BP 选择动态变化。

请在 engine.js 中修改 _play_next_beat 方法：

```javascript
_play_next_beat() {
  if (!this.is_playing) return;
  this._clear_timers();
  const scene = this.scenes[this.state.current_scene];
  if (!scene) return;

  // 支持 beats 条件跳过：condition 返回 false 的 beat 自动跳过
  while (this.state.beat_index < scene.beats.length) {
    const beat = scene.beats[this.state.beat_index];
    if (beat.condition && !beat.condition(this.state)) {
      this.state.beat_index++;
      continue;
    }
    this.state.beat_index++;
    this._execute_beat(beat);
    return;
  }

  this._on_beats_finished(scene);
}
````

## 关键设计说明

**为什么 beats 需要 condition？**

Galgame 的精髓就是**前面的选择影响后面的对话**。如果选了纳尔，后面所有提到"鳄鱼"的文本都必须变成"纳尔"，否则玩家会立刻出戏。

上面的方案是在 **不增加新场景 ID** 的前提下，用 `beat.condition` 让同一场景内的文本根据 flag 自动切换。这样：
- 不需要做 `ch1_010_croc` / `ch1_010_gnar` 两个场景
- test_runner 不会报死链
- 代码最干净


**关于内讧线触发：**

原条件 `(s.affinity.jackey || 0) < 30 && (s.affinity.coach || 0) < 30` 是死代码，因为目前没有任何选项能让 JackeyLove 好感降到 30 以下。改为 `coach < 40` 后，如果你走"装睡→顶嘴→BP 顶嘴"路线，白色月牙好感会降到 44 以下，内讧线就能触发，游戏才有**真正的多周目价值**。
